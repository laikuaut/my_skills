# interactions.md テンプレート（D1.5 マイクロインタラクション）

```markdown
# マイクロインタラクション (D1.5)

- 対象: アニメーション・フィードバック・キーボード操作
- 作成日: YYYY-MM-DD
- 関連: [components.md](./components.md) → 本書
- 状態: draft

## 1. 設計原則

- **150ms 以下の短い遷移**を基本。長いアニメーションは操作を妨げる
- **減速イージング**（ease-out）を既定とし、退出には ease-in
- **prefers-reduced-motion** を尊重し、無効化された環境ではアニメーションを止める

## 2. タイミング

| 名前 | 時間 | 用途 |
|------|------|------|
| instant | 0ms | 即時反応（チェック・トグル） |
| fast | 120ms | hover / focus |
| base | 200ms | モーダル開閉 |
| slow | 350ms | ページ遷移 |

## 3. 状態フィードバック

| 操作 | フィードバック |
|------|---------------|
| ボタン hover | 背景色を1段階明度変化（fast） |
| ボタン press | 縮小 scale(0.98)（instant） |
| 保存成功 | Toast を右上に表示、3秒で自動消去 |
| 削除確認 | Modal で確認、破壊操作は赤ボタン |
| 読込中 | ボタン内 spinner、画面全体ローディングは避ける |

## 4. キーボード操作

| 対象 | キー | 挙動 |
|------|------|------|
| モーダル | Esc | 閉じる |
| フォーム | Ctrl/Cmd + Enter | 送信 |
| テーブル | ↑ / ↓ | 行選択移動 |
| 検索ボックス | / | フォーカス |

## 5. アニメーション対象外

- テーブルの行追加・削除（レイアウト崩れを招く）
- 頻繁に更新される数値表示

## 6. 引継ぎメモ

- D2.3 state-management へ: Toast の保持は UI state（例: Zustand）、3秒タイマーで消去
- テスト（playwright-test）: waitForTimeout 禁止、`waitFor` でアニメーション完了を待つ

## 7. 完了判定 (DoD)

- [ ] タイミング表・状態フィードバック表・キーボード操作表が揃っている
- [ ] prefers-reduced-motion の扱いが明記
- [ ] アニメーション対象外が明示
```
