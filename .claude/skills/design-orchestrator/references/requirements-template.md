# requirements.md テンプレート

新規プロダクトの要求仕様書のひな形。採用/不採用はユーザに確認する。

```markdown
# {プロダクト名} 要求仕様 v1.0

最終更新: YYYY-MM-DD
ステータス: draft / confirmed / frozen

## 1. プロダクト概要

- **目的**: 1〜2行
- **想定ユーザー**: ペルソナを1〜3名
- **最優先価値**: UI/UX品質 / 機能数 / 性能 / プライバシー のどれか1つ

## 2. スコープ

### 2.1 含む（MVP）
- 機能A
- 機能B

### 2.2 含まない（次フェーズ以降）
- 機能X
- 機能Y

## 3. ユーザーストーリー

- US-01: {役割} として {目的} のために {機能} したい
- US-02: ...

## 4. 機能要件

### 4.1 基本機能
| ID | 機能 | 優先度 | 備考 |
|----|------|--------|------|
| F-01 | ログイン | 必須 | |
| F-02 | 一覧表示 | 必須 | 1画面25件以上 |

### 4.2 データ操作
- 作成 / 読取 / 更新 / 削除の粒度を明示

### 4.3 エクスポート・インポート
- 対応フォーマット（JSON / CSV / ...）
- 衝突解決方針（上書き / 新ID採番）

## 5. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| パフォーマンス | 初回描画 < 1s、操作応答 < 100ms |
| セキュリティ | 認証方式、保存データの暗号化有無 |
| プライバシー | 外部送信の有無、ログ保存ポリシー |
| i18n | 対応言語、切替方法 |
| アクセシビリティ | WCAG 2.1 AA / キーボード操作 |

## 6. 技術スタック

| 層 | 選定 |
|----|------|
| フロント | React + Vite + TypeScript + Tailwind |
| バック | FastAPI + Python 3.12 + uv |
| DB | SQLite / PostgreSQL / ファイル |
| 配信 | Docker + nginx / 静的ホスティング |
| MCP | 連携有無と対象 |

## 7. 運用・デプロイ

- ホスティング: ローカル / VPS / クラウド
- CI/CD: GitHub Actions / なし
- バックアップ: 要否と方針

## 8. データ保存方式

- 形式: DB / ファイル / IndexedDB
- スキーマのバージョニング
- マイグレーション方針

## 9. 初期データ・サンプル

- 初期状態は空 / サンプル同梱
- サンプル内容: 例）タグ4件 + タスク8件

## 10. 確定事項（サマリ）

| 項目 | 確定内容 | 確定日 | 出典 |
|------|----------|--------|------|
| 認証 | パスワード + メール | 2026-04-19 | Phase 1 AskUserQuestion |
| i18n | 日本語のみ（v1） | 2026-04-19 | 同上 |

### 10.1 根拠・影響範囲
- 認証方式を確定したことで、Phase 2 の D2.1 アーキで SSO を除外できる
- ...

## 11. 未決事項（チェックリスト）

- [ ] 画面モックの最終版（D1.4 で確定予定）
- [ ] MCP連携の要否（D4.x で確定予定）
- [ ] バックアップ頻度
```

## 使い方

- **採用しないセクション**はユーザに確認のうえ削除してよい
- 「§10 確定事項」と「§11 未決事項」は必ず残す。設計フェーズ中に双方を往復する
- 既存要求が変わったら **requirements.md を先に更新してから設計に反映**する
